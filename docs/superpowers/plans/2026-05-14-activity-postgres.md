# Activity tab → Postgres via GraphQL backend — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. **Do NOT use code snippets when generating implementation code based on this plan — the implementing agent writes the code. This plan describes specs, tests, contracts, and acceptance criteria only.**

**Goal:** Migrate the protocol details Activity tab off local SQLite and onto the tracker's Postgres database, served through the Rust GraphQL backend, with lazy-on-tab-click loading, cursor pagination, polling refresh, and on-demand detail fetch by hash.

**Architecture:** Add three columns to the tracker `matches` table (`repo_scope`, `repo_name`, `repo_version`) so we can filter by OCI repo identity rather than by the unreliable TII inner name. Expose two new GraphQL queries (`protocolMatches`, `protocolMatch`) from the existing Rust backend, backed by a fresh `sqlx::PgPool`. Replace the frontend's SQLite Kysely path with two resource routes consumed by the Activity tab via `useFetcher` plus a polling hook.

**Tech stack:** Rust (sqlx 0.8, async-graphql 7, Rocket 0.5, chrono, base64). TypeScript (React Router v7, graphql-request, GraphQL Codegen, React 19). PostgreSQL (BIGSERIAL, BYTEA, JSONB, TIMESTAMPTZ). Tests: `#[sqlx::test]` for tracker + backend; React Testing Library + Vitest for frontend.

**Reference spec:** `docs/superpowers/specs/2026-05-14-activity-postgres-design.md`.

**Branch:** all work commits to `feat/add-activity-tab`. Single PR at the end.

**Goal.** Provide a deterministic, opaque cursor format `base64("id:<row_id>")` with encode/decode helpers and a unit-tested codec.

**Implementation specs.**
- `encode_cursor(id: i64) -> String`: returns the base64 (standard alphabet, with padding) of UTF-8 bytes `id:<id>`.
- `decode_cursor(cursor: &str) -> Result<i64, CursorError>`: base64-decodes, parses the `id:<n>` prefix exactly, returns the `i64`. Any deviation (missing prefix, non-numeric, base64 error, extra trailing bytes) returns `CursorError::Malformed`.
- `CursorError` is `thiserror::Error`-friendly OR a plain enum with `Display` — implementing agent decides. Must map to a clean GraphQL error message ("invalid cursor") in the resolver.

**Tests** (`#[cfg(test)] mod tests` inside `cursor.rs`).
- `roundtrip_encode_decode`: encode several `i64` values (0, 1, 123, `i64::MAX`); decode returns the same value.
- `decode_rejects_malformed_base64`: input like `"not!base64$"` returns `Malformed`.
- `decode_rejects_missing_prefix`: input is valid base64 of `"42"` (no `id:`); returns `Malformed`.
- `decode_rejects_trailing_junk`: input is valid base64 of `"id:42xyz"`; returns `Malformed`.
- `decode_rejects_negative_id`: input is valid base64 of `"id:-1"`; returns `Malformed` (negative ids are nonsensical here).

## Task 5 — Backend DB helpers for matches

**Files:**
- Modify: `backend/src/db.rs` (extend with `MatchRow`, `fetch_matches`, `fetch_match`).
- Tests: `backend/tests/db_matches.rs` (new).

**Goal.** Two well-tested SQL helpers that the resolver can call without leaking SQL into the GraphQL layer. Pagination is a `LIMIT N+1` strategy so the caller can compute `hasNextPage` from one round-trip.

**Implementation specs — `MatchRow` (internal, NOT GraphQL):**
- Fields: `id: i64`, `tx_hash: Vec<u8>`, `block_slot: i64`, `block_hash: Vec<u8>`, `source_name: String`, `protocol_name: String`, `profile_name: String`, `tx_name: String`, `repo_scope: String`, `repo_name: String`, `repo_version: String`, `lifted: String` (read as text from the JSONB column; the resolver does not parse it), `matched_at: chrono::DateTime<chrono::Utc>`.

**Implementation specs — `fetch_matches`:**
- Signature: `pub async fn fetch_matches(pool: &PgPool, scope: &str, name: &str, version: Option<&str>, first: i64, after_id: Option<i64>) -> Result<(Vec<MatchRow>, bool), sqlx::Error>`.
- SQL contract: `SELECT id, tx_hash, block_slot, block_hash, source_name, protocol_name, profile_name, tx_name, repo_scope, repo_name, repo_version, lifted::text AS lifted, matched_at FROM matches WHERE repo_scope = $1 AND repo_name = $2 AND ($3::text IS NULL OR repo_version = $3) AND ($4::bigint IS NULL OR id < $4) ORDER BY id DESC LIMIT $5`.
- Bind `$5 = first + 1`. If the result set length equals `first + 1`, set `has_next = true` and drop the last row from the returned `Vec`. Otherwise `has_next = false`.
- `first` is clamped to `[1, 200]` by the caller (resolver); this helper does not re-clamp.

**Implementation specs — `fetch_match`:**
- Signature: `pub async fn fetch_match(pool: &PgPool, scope: &str, name: &str, tx_hash: &[u8]) -> Result<Option<MatchRow>, sqlx::Error>`.
- SQL: `SELECT ... FROM matches WHERE repo_scope = $1 AND repo_name = $2 AND tx_hash = $3 LIMIT 1`.
- Returns `None` if no row.

**Tests** (`backend/tests/db_matches.rs`, using `#[sqlx::test(migrations = "../tracker/migrations")]` so the test DB has the tracker schema applied).
- `fetch_matches_filters_by_scope_and_name`: insert 4 rows — 2 under (`txpipe`, `orcfax-burn`), 1 under (`txpipe`, `other`), 1 under (`other-scope`, `orcfax-burn`). Call with scope=`txpipe`, name=`orcfax-burn`, version=`None`, first=10, after=`None`. Expect 2 rows returned.
- `fetch_matches_orders_newest_first`: insert 3 rows in the same `(scope, name)` in known time order. Verify the returned `Vec` order is by descending `id`.
- `fetch_matches_filters_by_version`: insert rows under versions `1.0.0`, `2.0.0`, `2.0.0`. Filter version=`Some("2.0.0")`. Expect 2 rows.
- `fetch_matches_paginates_with_after_id`: insert 5 rows, take first 2 with no `after`, take next 2 with `after = id_of_oldest_in_first_page`. Verify the pages don't overlap and the union is 4 unique rows newest-to-oldest.
- `fetch_matches_has_next_when_more_remain`: insert 3 rows, request first=2. Expect `(2 rows, has_next=true)`.
- `fetch_matches_has_next_false_when_exhausted`: insert 2 rows, request first=5. Expect `(2 rows, has_next=false)`.
- `fetch_match_returns_row_by_hash`: insert a known row, fetch by `(scope, name, tx_hash)`, expect `Some(row)` with matching fields.
- `fetch_match_returns_none_when_missing`: fetch with a random 32-byte hash that wasn't inserted; expect `None`.
- `fetch_match_scope_isolation`: insert one row under (`txpipe`, `orcfax-burn`, tx_hash=X); calling `fetch_match` with scope=`other-scope` and the same `tx_hash` returns `None`.

## Task 7 — Resolver: `protocolMatches`

**Files:**
- Create: `backend/src/schema/match_query/query.rs`.
- Modify: `backend/src/schema/match_query/mod.rs` to declare `mod query;` and re-export `MatchQuery`.
- Modify: `backend/src/schema/mod.rs` to add `MatchQuery` into the `MergedObject` `Query`.
- Tests: `backend/tests/graphql_protocol_matches.rs` (new).

**Goal.** Top-level GraphQL query `protocolMatches(scope, name, version, first, after) -> MatchConnection`. Pagination uses the cursor codec; `first` is clamped to `[1, 200]` with default `50`.

**Implementation specs.**
- Resolver method on `MatchQuery`: `async fn protocol_matches(&self, ctx: &Context<'_>, scope: String, name: String, version: Option<String>, first: Option<i32>, after: Option<String>) -> Result<MatchConnection, Error>`.
- Default `first = 50`. Clamp to `[1, 200]`. Reject negative values with `Error::new("first must be >= 1")`.
- If `after` is `Some`, decode via `cursor::decode_cursor`. On `CursorError::Malformed` → `Error::new("invalid cursor")`.
- Call `db::fetch_matches(pool, &scope, &name, version.as_deref(), first as i64, after_id)`. The pool comes from `ctx.data_unchecked::<PgPool>()`.
- Build `MatchConnection`:
  - `nodes`: rows mapped through `From<MatchRow> for Match`.
  - `edges`: one per node, each `cursor = encode_cursor(row.id)` before the row is moved into the node.
  - `pageInfo.has_next_page = has_next` from `fetch_matches`. `pageInfo.has_previous_page = after.is_some()` (conservative; we don't currently support backwards paging). `pageInfo.start_cursor` = first edge's cursor or `None`. `pageInfo.end_cursor` = last edge's cursor or `None`.
- Map any `sqlx::Error` to `Error::new("database unavailable")` (log the underlying error via `tracing` if present, otherwise `eprintln!`).

**Tests.**
